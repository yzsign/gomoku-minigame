# 首页好友列表（收起 / 展开）— 需求总结

**文档版本**：V1.1  
**适用产品**：团团五子棋微信小游戏  
**核心模块**：好友列表（支持收起 / 展开）  
**需求来源**：基于前期好友列表功能、添加好友功能延伸；列表以侧边入口展开，悬浮于首页之上，优化用户交互体验  
**涉及端**：微信小程序前端、服务端  

**关联文档**：

- [对手战绩弹窗 · 添加好友](friend-request-social-spec.md)（好友申请、频控、WS 等）
- 仓库规则 `gomoku-social-friend`：`POST /api/social/friend-requests`、`GET /api/social/friend-status`、用户 WebSocket 下行类型等

---

## 一、核心需求总览

设计可收起 / 展开的好友列表，以侧边悬浮按钮为打开入口；列表展开后悬浮于首页之上，**允许遮挡**首页主功能区域（用户需收起列表后再操作首页入口）；保留好友列表核心社交能力，兼顾实用性与视觉美观，适配现有游戏 UI 风格，并实现**展开 / 收起状态**本地持久化。

---

## 二、核心功能要点

### 2.1 入口设计（重点）

| 项目 | 说明 |
|------|------|
| **打开入口** | 首页左侧中间区域，**侧边悬浮按钮**：圆形 **48×48px**，背景与首页主题色一致（浅米色），图标为深棕色好友轮廓；点击后展开列表，**展开后该悬浮按钮隐藏**。 |
| **收起入口** | ① 好友列表**顶部标题栏右侧**文字按钮「收起」：**14px**，默认灰色，**交互态**为深棕色（见 **§5.3 平台说明**）。② 支持点击**列表外区域**一键收起（见 **§2.2**），收起后悬浮按钮重新显示。 |

### 2.2 收起 / 展开交互

- **展开**：点击侧边悬浮按钮，列表从左侧滑入，伴随 **300ms** 淡入动画（`ease-in-out` 缓动），悬浮于首页之上。
- **收起**：点击「收起」或 **列表外可点击区域**（见下），列表向左侧滑出，伴随 **300ms** 淡出动画，悬浮按钮恢复显示。
- **点击收起范围（已定）**：
  - 展开时应有**半透明遮罩**覆盖首页（列表可与遮罩一并盖住主功能区域）；点击**遮罩**（列表面板以外的区域）即收起。
  - 不要求对首页主功能区做「点击穿透」：**展开期间**以列表 + 遮罩为顶层，用户通过「收起」或点遮罩关闭后再操作首页。

- **状态持久化**：收起 / 展开状态写入**本地缓存**（如 `wx.setStorage`），重新进入首页时恢复上次状态。

### 2.3 好友列表核心功能

- **好友展示**：头像圆形 **40×40px**；昵称 **14px** 深棕色，单行溢出省略；在线状态：**绿色 = 在线**，**灰色 = 离线**。
- **好友操作**：好友对战（**仅在线好友**可点）、发起聊天、删除好友、修改备注。  
  - **发起聊天**：**不接入**现有游戏内对局 / 房间会话；采用**独立方案**（如独立私聊页、半屏会话或产品另定的入口与协议），在专门需求或附录中定义路由与后端，与局内聊天模块**不复用**。
- **辅助功能**：搜索框（按昵称搜索）、分组切换（**全部好友 / 在线好友**）、空状态（「暂无好友」文案）。

### 2.4 与服务端 / 现有能力对齐

- **已实现（参考）**：好友申请创建 / 处理、好友状态查询、用户 WS（`FRIEND_REQUEST_*`、`FRIENDSHIP_UPDATED`）、对同一人 **24 小时内最多 3 次**申请等，见 [friend-request-social-spec.md](friend-request-social-spec.md)。
- **列表能力需产品 / 接口对齐**：首页列表依赖服务端提供（或补充）**好友列表、在线状态、备注读写、删除好友**等；**私聊**若本期要做，接口与客户端路由**独立建设**，**不**走现有局内会话通道。若部分能力分期交付，需在迭代说明中列出 **MVP 范围**。

## 三、UI 与交互规范

### 3.1 色彩

延续首页风格：**主色**浅米色 `#F5EADF`、**深棕色** `#5D4037`；**辅助色**绿色（在线）、灰色（离线 / 次要文字）。

### 3.2 尺寸（以 iPhone 14 为基准：375×812）

| 元素 | 规格 |
|------|------|
| 侧边悬浮按钮 | **48×48px**，位置约 **(10, 300)**，圆角 **24px**（圆形） |
| 展开后列表 | **280×580px**，位置约 **(10, 120)**，圆角 **12px**，白底 **透明度约 90%**，轻微阴影 |

### 3.3 动效

- 悬浮按钮：点击缩放 **0.95 → 1.05**，**150ms**。
- 好友状态刷新：**200ms** 淡入淡出。
- 列表展开 / 收起：滑入滑出 + 淡入淡出 **300ms**。

### 3.4 平台说明（微信小程序）

- **无桌面 `hover`**：文档中「hover 时深棕色」**落实为**：「收起」按钮的**按下态 / 高亮态**为深棕色，或与设计稿统一的激活态，**不得依赖 hover**。

## 四、关键约束

### 4.1 与首页主功能区的关系

- **允许遮挡**：列表展开时**可以**盖住首页主功能区域（含随机匹配、好友对战、人机对战、每日签到、我的战绩、杂货铺等），属预期表现。
- **交互含义**：展开期间用户优先完成好友相关操作；需使用首页入口时，应先**收起**列表（「收起」或点遮罩），再点击首页按钮。

### 4.2 防骚扰与数量

- 好友列表**最多展示 50** 个好友条目，超出部分**滚动加载**或滚动浏览（与实现一致即可）。
- 同一玩家对同一对手 **24 小时内最多 3 次**好友申请（与现有服务端频控一致）。

### 4.3 异常与同步

- **网络异常**：优先展示**本地缓存**的好友列表（若有），并 Toast / 轻提示网络异常。
- **好友关系变更**（删除 / 添加）：列表应**刷新**；可与用户 WS `FRIENDSHIP_UPDATED` 等事件及重新进入首页时的 HTTP 拉取结合。

---

## 五、验收要点

1. 侧边打开按钮、列表「收起」、点击遮罩 / 列表外收起行为正确；展开 / 收起动画流畅；**本地持久化**生效。
2. 列表展开后可遮挡首页主功能区；收起后主功能可正常点击；好友信息、操作按钮可用；在线 / 离线、好友关系与后端一致。
3. UI 与首页统一，无明显错位、卡顿、崩溃；空状态、网络异常展示符合 **§4.3**。
4. 小程序端交互符合 **§3.4**（无 hover 依赖）。
